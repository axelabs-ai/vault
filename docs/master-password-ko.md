# Master Password 가이드 (한국어)

Vaultwarden 셀프호스트의 마스터 패스워드는 여러분이 가진 **모든 비밀번호의 최종 열쇠**입니다. 잃어버리면 vault 전체를 잃고, 유출되면 vault 전체가 털립니다. 따라서 다른 사이트 비밀번호와는 다른 강한 원칙으로 만듭니다.

---

## 1. 왜 패스프레이즈인가 — 짧은 암호 vs 단어 조합

전통적인 12자 비밀번호 `Tr0ub4dor&3` 같은 형태는 외우기 어렵고, 의외로 강하지도 않습니다.

| 방식 | 예시 형태 | 외우기 | 입력 | 엔트로피 |
|---|---|---|---|---|
| 12자 복잡 암호 | `Tr0ub4dor&3` | 어렵다 | 오타 잦음 | 약 30~40 bits |
| 6단어 영문 패스프레이즈 | 단어-단어-단어-단어-단어-단어 | 시각화 가능 | 의외로 빠름 | 약 77.5 bits |
| 5단어 한국어 패스프레이즈 | 단어-단어-단어-단어-단어 | 한국어 모국어자에게 매우 쉬움 | 한영 전환 필요 | 약 60 bits |

**핵심 원리**: 패스프레이즈는 단어 풀(7776개 또는 4096개)에서 균일 무작위로 뽑은 단어를 이어 붙입니다. 각 단어는 독립적인 무작위 선택이므로 엔트로피가 정확히 계산됩니다. 사람이 "복잡해 보이게" 만든 암호는 추측 가능한 패턴(첫 글자 대문자, 끝에 숫자, 흔한 단어 변형)으로 가득 차서 실제 엔트로피가 낮습니다.

> XKCD 936 만화 참고: "복잡해 보이는 짧은 암호보다 단순해 보이는 긴 단어 조합이 강하다."

---

## 2. 본인용 (영문 6단어, ~77.5 bits)

운영자(법인 주인) 본인은 vault의 마지막 방어선입니다. 가장 강한 마스터 패스워드를 씁니다.

### 생성
```bash
~/vault/scripts/diceware-en.sh
```

- EFF Long Wordlist (7776 단어)에서 6단어 무작위 선택.
- 5개 후보가 출력됩니다. 그중 **한 장면으로 시각화할 수 있는** 조합 하나를 고르세요.
- 예시 (실제 사용 금지): `panther-kiwi-velcro-saddle-mango-thicket` → "팬더가 키위 농장 옆 벨크로 안장에 망고를 매단 채 덤불을 지나간다."

### 엔트로피
- 6 × log₂(7776) ≈ 77.5 bits
- 추측 공간 ≈ 4 × 10²³ 회
- Bitwarden 기본 KDF (PBKDF2-600k 또는 Argon2id)로 보호되면, 현실적 공격 비용은 사실상 불가능.

## 3. 가족용 (한국어 4~5단어)

가족(부모님, 배우자, 자녀)에게는 한국어 단어 조합이 훨씬 외우기 쉽습니다.

### 생성
```bash
~/vault/scripts/diceware-ko.sh
```

- 직접 큐레이션한 4096개 한국어 명사 풀에서 무작위 선택.
- 4단어 패스프레이즈와 5단어 패스프레이즈 각각 5개씩 출력.

### 엔트로피
- 4단어 = 48 bits (단순 GPU 해시 공격 시 약 1개월에 깨질 수 있음 — 비추)
- 5단어 = 60 bits (KDF 보호 시 약 3600년 이상 — **권장**)

가족 마스터 패스워드는 **5단어 권장**.

### 외우는 법 — 시각화 한 장면

4~5단어를 하나의 **이상한 장면**으로 머릿속에 그립니다. 평범한 장면은 잊혀지고, 이상한 장면은 오래 기억됩니다.

예시 (생성된 단어 그대로 사용 금지 — 외우기 기법 설명용):
- 후보 단어: `푸른` `감자` `책상` `여덟`
- 시각화: "푸른색 감자 여덟 개가 책상 위에 줄지어 놓여 있고, 그중 하나가 굴러떨어진다."
- 5단어로 확장: + `우산` → "푸른 감자 여덟 개가 책상에 있고, 굴러떨어진 하나를 우산으로 받았다."

가족 구성원마다 본인의 장면을 직접 만들도록 합니다. 다른 사람이 만들어주면 외우지 못합니다.

## 4. 종이 백업 — 2부 분산 보관

마스터 패스워드는 어떤 디지털 매체에도 저장하지 않습니다. **종이에 손글씨로** 2부 만듭니다.

| 사본 | 보관 장소 |
|---|---|
| 1부 | 가정 금고 (방염·내수) |
| 2부 | 은행 대여금고 또는 신뢰할 수 있는 다른 물리적 장소 |

### 작성 규칙
- 손글씨로 또박또박. 인쇄·복사 금지(프린터 메모리·복사기 잔상 위험).
- 한 장에 마스터 패스워드만 단독 기재. "Bitwarden", "vault" 등 식별 단어 적지 않음.
- 봉투에 밀봉. 봉투 위에 무관한 표식만 (예: "○○ 보험 서류").
- 작성일과 작성자 이니셜만 작은 글씨로 한쪽 구석에.

### 옵션 — 금속 백업
화재·수해 시 종이는 손실됩니다. 더 견고한 백업이 필요하면:
- **Cryptosteel Capsule** (~$80)
- **Billfodl** (~$80)

스테인리스 강판에 문자를 박아 보관. 단어를 약자(첫 글자) 형태로 박으면 단어 수만큼만 표현 가능 — 단어 전체를 저장하려면 더 큰 모델 필요.

## 5. 분실 복구

마스터 패스워드를 잊었을 때의 복구 경로:

1. **Recovery Code** (Bitwarden 계정 설정에서 발급)
   - 발급 시점에 종이에 손글씨로 따로 받아둡니다.
   - Recovery Code는 마스터 패스워드와 **별도 장소**에 보관 (금고 다른 칸 또는 다른 봉투).
   - Recovery Code로 vault에 접근 → 마스터 패스워드 재설정 가능.

2. **로컬 백업 복원**
   - `~/vault/scripts/backup.sh`로 매일 백업이 B2에 암호화 업로드됩니다 (GPG 대칭키).
   - GPG 패스프레이즈는 마스터 패스워드와 **다른 비밀**로 관리합니다 (`~/vault/scripts/gpg-init.sh` 참조).
   - 신규 서버에 vault 복원 후 새 마스터 패스워드로 재시작.

3. **2단계 인증 백업 코드**
   - 2FA 활성 시 백업 코드도 종이로 받아둡니다.

## 6. 회전 정책

**원칙: 강제 회전 없음**

- 강한 마스터 패스워드는 회전 자체가 약점을 만듭니다 (새 암호 외우기 어려움 → 메모 유혹).
- NIST SP 800-63B도 정기 회전을 권장하지 않습니다.
- **회전이 필요한 경우**:
  - 마스터 패스워드를 어딘가에 타이핑한 의심이 들 때
  - 종이 백업이 분실·도난 의심
  - 가족 구성원과 공유한 디바이스에서 입력했을 가능성
  - vault에 비정상 접근 흔적이 보일 때 (Bitwarden 로그인 알림)

## 7. 절대 금지 사항

마스터 패스워드는 **다음 어디에도 입력하지 않습니다**:

- 메모 앱 (Apple Notes, Google Keep, Notion, Bear, Obsidian, ...)
- 이메일 (본문, 제목, 임시저장 모두)
- 메신저 (카톡, 텔레그램, 슬랙, 디스코드, ...)
- 클라우드 문서 (Google Docs, 한컴 클라우드, OneDrive, ...)
- 스프레드시트 (어떤 형태든)
- 캘린더/할일 앱 메모란
- 비밀번호 매니저의 보조 vault (Bitwarden의 다른 항목 포함)
- 코드 주석, README, .env 파일
- 클립보드 (복사 후 즉시 다른 텍스트로 덮어쓰기)
- 스크린샷 / 사진
- 음성 메모 / 녹음
- 채팅 AI (ChatGPT, Claude, Gemini, ...)
- 검색창 (실수로 주소창에 입력하면 검색 로그에 남음)

### 입력해도 되는 곳
- Bitwarden / Vaultwarden 로그인 화면 (공식 클라이언트, 공식 도메인)
- 모바일 Bitwarden 앱
- 본인 또는 가족의 머릿속

## 8. 체크리스트

새 마스터 패스워드를 셋업할 때:

- [ ] `diceware-en.sh` 또는 `diceware-ko.sh` 실행 → 후보 5개 중 1개 선택
- [ ] 시각화 장면 만들어 머릿속에 고정 (최소 3일간 매일 회상)
- [ ] 종이 사본 2부 손글씨 작성 (인쇄 금지)
- [ ] 사본 1: 가정 금고 보관
- [ ] 사본 2: 은행 대여금고 또는 분산 장소 보관
- [ ] Bitwarden 계정 설정에서 Recovery Code 발급 → 종이로 따로 보관
- [ ] 2FA 활성화 + 백업 코드 종이 보관
- [ ] 터미널 스크롤 버퍼 클리어 (`Cmd+K` on macOS Terminal)
- [ ] 발생한 임시 파일/메모 없는지 재확인
