# 문제 해결 가이드 — 우리 가족 vault

vault를 쓰시면서 자주 일어나는 문제를 모아 둔 문서입니다.
"고장 났네"하고 포기하지 마시고 이 문서를 먼저 한 번 훑어 보세요.
대부분 5분 안에 해결됩니다.

> 이 문서로도 안 풀리면 운영자에게 연락하세요. "vault troubleshooting 봤는데 안 됨"이라고 한 마디면 됩니다.

먼저 알아 두실 점: 우리 가족 vault가 작동하려면 **세 가지 다리**가 모두 잘 연결되어 있어야 해요.

1. **Tailscale** — 우리 집 가상 네트워크. 외부에서 우리 집 안으로 들어오는 다리.
2. **Bitwarden 앱** — 비밀번호를 보여 주는 도우미.
3. **macmini** — 비밀번호가 실제로 들어 있는 우리 집 금고.

문제가 생기면 보통 셋 중 하나가 끊어진 거예요. 차례로 확인합시다.

---

## 문제 1. Tailscale 연결 안 됨

### 증상

- Bitwarden 앱이 "Cannot reach server" 또는 "서버 연결 실패" 표시.
- 브라우저에서 vault 주소 입력하면 "사이트에 연결할 수 없음" 표시.
- 그러나 다른 사이트(네이버, 구글)는 잘 됨.

### 원인

폰·노트북에서 우리 집 안으로 들어오는 다리(Tailscale)가 끊어진 상태.

### 해결 (순서대로)

**1단계: Tailscale 앱이 켜져 있는지 확인**

- **아이폰/안드로이드**: Tailscale 앱 열기 → 상단의 토글 스위치가 **초록색**인지 확인.
  꺼져 있으면 한 번 누르기.
- **Mac**: 상단 메뉴 바(시계 옆) Tailscale 아이콘 클릭 → 메뉴에서 "Connected" 또는 본인 디바이스 이름이 보이는지.
  꺼져 있으면 "Log in" 또는 "Connect".
- **Windows**: 우측 작업 표시줄 트레이 Tailscale 아이콘 더블 클릭 → 상태 확인.

![](screenshots/trouble-01-tailscale.png)

**2단계: 다른 디바이스가 보이는지 확인**

Tailscale 앱 안에 "macmini"라는 디바이스가 보여야 합니다. 보이면 OK.

안 보이면:
- 운영자에게 연락 → 본인 디바이스가 우리 집 Tailscale 네트워크에 **승인** 안 되어 있을 수 있음.

**3단계: 폰을 비행기 모드로 켰다가 끄기 (LTE인 경우)**

가끔 LTE 네트워크와 Tailscale이 충돌합니다. 비행기 모드 5초 켰다가 끄면 풀립니다.

**4단계: Tailscale 앱 재시작**

- 폰: 앱 종료(앱 전환 → 위로 스와이프) → 다시 열기.
- 노트북: 메뉴에서 "Quit" → 다시 실행.

**5단계: 그래도 안 되면**

운영자에게 다음 두 가지 정보와 함께 연락하세요:
- "Tailscale 앱은 켜져 있고 macmini도 디바이스 목록에 보이는데 안 됨"
- Bitwarden 앱이 표시하는 정확한 에러 메시지 (사진 첨부).

## 문제 2. Bitwarden 앱이 "Cannot reach server"

### 증상

Bitwarden 앱 로그인 시 또는 동기화 시 "Cannot reach server" 또는 "서버에 도달할 수 없음".

### 원인

세 가지 가능성이 있습니다.

1. Tailscale이 꺼져 있음 → 문제 1로 가세요.
2. 서버 URL을 잘못 입력함.
3. macmini 본체가 꺼져 있음.

### 해결

**1단계: 서버 URL 다시 확인**

1. Bitwarden 앱 → 좌상단 톱니바퀴 또는 "지역" → **Self-hosted** 화면.
2. Server URL 칸 확인. **정확히** 다음과 같아야 함:
   ```
   https://macmini.<우리집이름>.ts.net
   ```
3. 흔한 실수:
   - `http://` (s가 빠짐) — 안 됨. `https://` 이어야 함.
   - 끝에 `/` 추가 — 안 됨. 슬래시 없이 끝.
   - 공백이 들어감 — 안 됨. 운영자가 보낸 주소를 다시 복사·붙여넣기.
4. 저장 → 다시 로그인 시도.

![](screenshots/trouble-02-url.png)

**2단계: macmini가 켜져 있는지 운영자에게 확인**

폰·노트북 다 OK인데 그래도 안 되면 macmini 자체가 꺼졌거나 정전·재부팅 중일 수 있음.
운영자에게 "vault 서버 살아 있나요?" 한 마디.

## 문제 3. 자동 입력이 일부 사이트에서만 작동

### 증상

네이버는 자동 입력 잘 되는데, 어느 특정 사이트(예: 어떤 쇼핑몰)에서는 안 됨.

### 원인

vault에 저장된 URL과 실제 사이트 URL이 안 맞음.

### 해결

**1단계: vault에서 해당 항목의 URL 확인**

Bitwarden 앱 열기 → 그 사이트 항목 찾기 → 편집 → "URL" 칸 확인.

예를 들어 vault에 `naver.com`만 저장되어 있는데, 실제 로그인 페이지는 `nid.naver.com`이면 자동 입력 안 됨.

**2단계: URL 추가**

같은 항목에 URL을 **여러 개** 추가할 수 있어요. + 버튼 눌러서:
- `naver.com`
- `nid.naver.com`
- `www.naver.com`

이렇게 다 추가하면 모든 페이지에서 자동 입력이 됩니다.

**3단계: URL "일치 감지" 모드 변경 (고급)**

그래도 안 되면 항목 편집 → URL 칸 옆 톱니바퀴 → **일치 감지(URL Match Detection)** 변경.

- **기본**: 도메인이 같으면 작동 (대부분 이걸로 충분).
- **호스트**: 같은 호스트네임(예: `nid.naver.com`)만.
- **시작 일치**: 주소가 같은 글자로 시작하면.

잘 모르시면 **기본(Base domain)**으로 두세요.

![](screenshots/trouble-03-url-match.png)

## 문제 4. 사이트에서 비밀번호를 바꿨는데 vault에 반영 안 됨

### 증상

은행 사이트에서 비밀번호 변경했는데, 다음에 자동 입력 시 옛날 비밀번호가 들어가서 로그인 실패.

### 원인

vault는 사이트 비밀번호 변경을 자동으로 감지하지 못합니다.
브라우저가 "업데이트할까요?" 물었을 때 본인이 못 보고 지나간 거예요.

### 해결

**방법 1: 수동 업데이트 (가장 확실)**

1. Bitwarden 앱 열기 → 해당 항목 찾기 → **편집**.
2. 비밀번호 칸에 새 비밀번호 직접 입력.
3. **저장**.

**방법 2: 다음 로그인 시 업데이트**

1. 그 사이트에 새 비밀번호로 직접 로그인.
2. 브라우저 자물쇠 아이콘이 "이 비밀번호로 업데이트할까요?" 물어봄 → **업데이트**.

> 옛날 비밀번호도 며칠 동안 메모해 두세요. 어떤 사이트는 비밀번호 변경 후 며칠 안에 옛날 비밀번호로 되돌릴 수 있는 "취소" 기능이 있어요.

## 문제 5. 한국어 입력이 안 됨 (Master Password 칸)

### 증상

Bitwarden 첫 로그인 시 Master Password를 한국어로 만드셨는데, 입력이 영어로만 됨.

### 원인

폰·노트북의 키보드 언어가 영어로 잠겨 있음.

### 해결

**아이폰**

1. Master Password 입력 칸에 커서를 둠.
2. 키보드 좌하단 지구본(🌐) 아이콘 누르기 → 한국어 키보드 선택.
3. 입력.

**안드로이드**

1. 키보드 우하단 톱니바퀴 또는 지구본 → 한국어 선택.
2. 입력.

**Mac**

1. 우측 상단 메뉴 바의 입력 메뉴(한·A) 클릭 → 한국어 선택.
2. 또는 Caps Lock 누르면 토글됨.

**Windows**

1. 우측 작업 표시줄의 "한" 또는 "A" 표시 클릭 → 한국어.
2. 또는 한/영 키 누르기.

### 예방

다음번부터는 Master Password를 입력할 때 키보드 언어를 **먼저 확인**하세요.

## 문제 6. vault 동기화가 느림

### 증상

노트북에서 비밀번호 추가했는데, 폰에는 한참 후에 나타남.

### 원인

Bitwarden 앱은 일정 주기로만 자동 동기화를 합니다. 즉시 보고 싶으시면 수동.

### 해결

**수동 동기화**

- Bitwarden 앱 → 좌하단 또는 우하단 **설정** 탭 → **동기화** → "지금 동기화" 누름.

**자동 동기화 주기 변경**

- 설정 → **동기화** → "동기화 간격"을 5분, 1시간 등으로 조정.

기본은 시간당 한 번 정도라 약간 느립니다. 자주 쓰시면 5분으로 줄여도 OK.

> 다만 너무 자주 하면 배터리가 빨리 닳습니다. 30분 정도가 무난.

## 문제 7. 아이폰 자동 채우기 vs Bitwarden 직접 자동 채우기 — 뭐가 다른가요?

### 헷갈리기 쉬운 점

아이폰에는 두 가지 자동 채우기가 있어요.

1. **iOS 시스템 자동 채우기** — 폰 설정 → 암호 → 암호 자동 완성 → Bitwarden.
2. **Bitwarden 앱 직접 자동 채우기** — Bitwarden 앱을 직접 열어서 복사·붙여넣기.

대부분은 1번이 편합니다. 자동으로 입력 칸에 채워 주거든요.

### 권장 설정

- **iOS 시스템 자동 채우기**: 켜기. **Bitwarden만 체크**, iCloud 키체인 끄기.
- **Bitwarden 앱**: 평소 비밀번호 보고 싶을 때만 열기.

iCloud 키체인을 켜 두면 두 가지가 동시에 떠서 어느 게 진짜인지 헷갈립니다.

### 둘 다 안 될 때

1. iOS 설정 → 암호 → 암호 옵션 → 모든 설정 다시 확인.
2. Bitwarden 앱이 잠금 해제 상태인지 (잠겨 있으면 시스템에서도 자동 입력 안 됨).
3. 폰 재부팅 (전원 버튼 길게 누르기 → 끄기 → 30초 후 다시 켜기).

## 문제 8. Mac Safari 확장이 작동 안 함

### 증상

Safari 우측 상단에 Bitwarden 자물쇠 아이콘이 안 보이거나, 눌러도 반응 없음.

### 해결

**1단계: 확장이 켜져 있는지 확인**

1. Safari → 상단 메뉴 **Safari** → **환경설정** → **확장 프로그램** 탭.
2. **Bitwarden** 옆 체크박스가 켜져 있는지 확인.
3. 안 보이면 Mac App Store에서 "Bitwarden for Safari" 검색해서 설치.

![](screenshots/trouble-04-safari-ext.png)

**2단계: 도구 모음에 추가**

1. 같은 환경설정 → **도구 모음** 또는 **사용자 설정** 탭.
2. **도구 모음 사용자 설정**으로 이동.
3. Bitwarden 아이콘을 드래그해서 도구 모음에 놓기.

**3단계: Safari 재시작**

- Safari 종료 (cmd+Q) → 다시 열기.

**4단계: Mac 재부팅**

- 위 단계 모두 시도해도 안 되면 Mac 재부팅 한 번.

**5단계: 확장 재설치**

- Safari 환경설정 → 확장 → Bitwarden 우클릭 → "제거".
- Mac App Store에서 다시 설치.

## 문제 9. 항목을 실수로 지웠어요

### 증상

비밀번호 항목을 삭제했는데 필요해졌어요.

### 해결

vault는 삭제된 항목을 **30일 동안 휴지통에 보관**합니다.

1. Bitwarden 앱 → 좌하단 또는 우상단 메뉴 → **휴지통(Trash)**.
2. 삭제한 항목 찾기 → 누름 → **복원**.
3. 다시 본인 금고로 돌아옴.

30일이 지나면 영구 삭제되어 복구 불가. 그래도 가족 공유 항목이면 다른 가족이 갖고 있을 수 있으니 운영자에게 확인.

## 문제 10. 가족 공유 컬렉션이 안 보임

### 증상

운영자가 가족 공유 컬렉션에 추가해 줬다는데, 본인 vault에는 안 보임.

### 해결

**1단계: 동기화**

위 문제 6 참조. 수동 동기화 한 번.

**2단계: 컬렉션 메뉴 확인**

Bitwarden 앱 금고 화면 → 상단 **모든 항목**을 누르고 → **컬렉션** 또는 **조직(Organization)** 메뉴 → **가족 공유** 보이는지.

**3단계: 운영자에게 확인**

- "운영자님, 저 가족 공유 컬렉션에 잘 추가되었나요?"
- 운영자가 vault 관리자 페이지에서 확인 가능.

## 그래도 안 풀릴 때 — 운영자에게 연락하는 법

다음과 같이 정리해서 보내 주시면 운영자가 빨리 도와드립니다.

```
[증상]
어떤 상황에서 무엇이 안 되는지.
예: "아이폰에서 Bitwarden 앱 열면 동기화가 멈춰 있어요."

[방금 한 일]
지금까지 무엇을 시도해 봤는지.
예: "Tailscale 껐다 켰고, Bitwarden 앱 재시작했고,
폰도 재부팅했어요."

[화면 사진]
가능하면 화면 사진 1~2장 첨부.
(Master Password가 보이지 않게!)

[디바이스]
어느 폰·노트북에서 일어났는지.
예: "아이폰 14, iOS 17.4"
```

부끄러워하지 마세요. IT 잘 모르신다고 운영자가 한심하게 보지 않습니다.
오히려 물어봐 주시는 게 도움이 됩니다.

## 자가 진단 체크리스트

다음 7가지를 확인하시면 90%의 문제는 해결됩니다.

- [ ] Tailscale 앱이 켜져 있고 "Connected" 상태인가?
- [ ] Wi-Fi 또는 LTE에 연결되어 있는가?
- [ ] Bitwarden 앱에 입력한 서버 URL이 정확한가? (`https://`로 시작, 끝에 `/` 없음)
- [ ] Bitwarden 앱이 잠금 해제 상태인가?
- [ ] iOS/Android 자동 채우기 설정에서 Bitwarden이 체크되어 있는가?
- [ ] vault 항목의 URL이 실제 사이트 주소와 일치하는가?
- [ ] 마지막 동기화가 언제인지 확인 (수동 동기화 한 번)?

이 7가지 다 OK인데 안 되면, 그땐 정말 운영자에게 연락해 주세요.
